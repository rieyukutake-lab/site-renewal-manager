# 🚀 Netlifyデプロイ手順

## 必要なファイル

以下のファイルとフォルダをすべてダウンロードしてください：

```
プロジェクトフォルダ/
├── index.html
├── netlify.toml
├── README.md
├── css/
│   └── style.css
└── js/
    ├── app.js
    └── auth.js
```

## ステップ1: ファイルのダウンロード

1. このエディタから上記すべてのファイルをダウンロード
2. ローカルPCに同じフォルダ構造で保存
3. フォルダ構造が正しいことを確認

## ステップ2: Netlifyアカウント作成

1. https://www.netlify.com/ にアクセス
2. 「Sign up」をクリック
3. 以下のいずれかで登録（無料）：
   - GitHubアカウント（おすすめ）
   - GitLabアカウント
   - Bitbucketアカウント
   - メールアドレス

## ステップ3: サイトのデプロイ

### 方法A: ドラッグ&ドロップ（最も簡単）

1. Netlifyにログイン
2. ダッシュボードで「**Add new site**」をクリック
3. 「**Deploy manually**」を選択
4. **フォルダごと**ドラッグ&ドロップエリアにドロップ
   - ⚠️ 重要: 個別ファイルではなく、プロジェクトフォルダ全体をドロップ
5. 自動的にデプロイが開始されます（数秒〜1分）
6. 完了すると `https://ランダムな名前.netlify.app` のURLが発行されます

### 方法B: Netlify CLI（技術者向け）

```bash
# Node.jsがインストール済みの場合
npm install -g netlify-cli
netlify login
netlify deploy --prod
```

## ステップ4: サイト名の変更（オプション）

1. Netlifyダッシュボードでサイトを開く
2. 「Site settings」→「Site details」
3. 「Change site name」をクリック
4. わかりやすい名前に変更（例: `my-site-renewal-manager`）
5. URLが `https://my-site-renewal-manager.netlify.app` に変更されます

## ステップ5: 動作確認

発行されたURLにアクセスして確認：

1. ✅ ログイン画面が表示される
2. ✅ パスワード「renewal2024」でログインできる
3. ✅ デザインが正しく表示される
4. ✅ 「新規追加」ボタンが動作する
5. ✅ 項目の登録・編集・削除ができる

## ⚠️ 重要な注意事項

### データベースAPIについて

このアプリは `tables/issues` というAPIエンドポイントを使用しています。

**確認が必要：**
- 現在の開発環境（Genspark）のAPIがNetlifyからアクセスできるか
- APIエンドポイントのURLを設定する必要があるか

**もしデータ保存が動作しない場合：**

1. ブラウザの開発者ツール（F12）を開く
2. Consoleタブでエラーメッセージを確認
3. エラー内容を教えていただければ対処方法を案内します

**代替案：**
- Supabase（無料のデータベースサービス）への移行
- Firebase Firestoreの利用
- ローカルストレージを使った簡易版

## ステップ6: クライアントへの共有

デプロイが成功したら、以下の情報をクライアントに共有：

```
【修正管理表アプリのご案内】

■ アクセスURL
https://あなたのサイト名.netlify.app

■ ログインパスワード
renewal2024

■ 使い方
1. URLにアクセス
2. パスワードを入力してログイン
3. 「新規追加」で修正項目を登録
4. 画像はドラッグ&ドロップで簡単登録
5. ステータスを更新して進捗管理
```

## 更新方法

アプリを更新したい場合：

1. ファイルを修正
2. Netlifyダッシュボードの「Deploys」タブ
3. 更新したファイルをドラッグ&ドロップ
4. 自動的に再デプロイされます

## トラブルシューティング

### CSSが効かない
- フォルダ構造が正しいか確認
- `css/style.css` が正しい場所にあるか確認

### JavaScriptが動作しない
- ブラウザコンソールでエラーを確認
- `js/app.js` と `js/auth.js` が正しい場所にあるか確認

### データが保存されない
- APIエンドポイントの設定が必要な可能性
- 開発者ツールでネットワークエラーを確認

## サポート

問題が発生した場合は、以下の情報を教えてください：

1. 発行されたNetlify URL
2. ブラウザのコンソールエラー（F12で確認）
3. どの機能で問題が起きているか
